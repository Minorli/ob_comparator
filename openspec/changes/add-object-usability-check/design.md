## Context
迁移后期发现“对象存在但不可用”的问题频发，尤其是 VIEW 与 SYNONYM（FORCE 创建、依赖缺失、权限不足、同义词链断裂）。现有对比仅检查存在性，无法覆盖此类问题。

## Goals / Non-Goals
- Goals:
  - 提供可选的 VIEW/SYNONYM 可用性校验
  - 提供源端对照与清晰的可用性报告
  - 支持并发、超时、采样控制，避免拖慢主流程
- Non-Goals:
  - 不对 TABLE/PLSQL/TRIGGER 做运行时可用性执行
  - 不保证捕获所有运行时逻辑错误（仅静态可解析可用性）

## Decisions
- 校验方式：`SELECT * FROM <obj> WHERE 1=2`，仅触发解析与依赖校验。
- 超时：单对象超时，超时记为 TIMEOUT，不判定为不可用。
- 并发：线程池并发（I/O bound），默认 10。
- 采样：可选能力，默认关闭；当显式配置阈值与比例时才启用抽样。
- 输出：独立明细报告 + 主报告汇总。

## Risks / Trade-offs
- 风险：对象数量大时耗时增多；通过超时/并发/采样控制。
- 风险：权限不足导致误判；报告中显式标记 ORA-01031 并给出建议。

## Migration Plan
- 新增开关默认关闭，不影响现有运行。
- 开启后逐步观察耗时与准确性。

## Open Questions

- 是否对 VIEW/SYNONYM 采用分类型独立超时策略？
